## 4. index.html 섹션 구성

순서대로:

1. **상단 고정 네비게이션 (sticky)**
   - 브랜드: `● 교무실 옆 작업실` (왼쪽, 빨간 점 + 고운바탕체)
   - 링크 5개: `홈 / 공개 도구 / 교내 대시보드 / 작업일지 / 연락`
   - 노트 페이지에서는 `작업일지` 링크가 active 표시

2. **섹션 1 — 홈 (`#home`)**
   - 베이지 배경
   - 헤더: 브랜드마크 ("교사가 · 교사들을 위해 · 취미로") + 메인 타이틀 + 서브타이틀
   - 인트로 박스: "이거 내가 직접 짜볼 수 있을까?…" 시작하는 글

3. **섹션 2 — 공개 도구 (`#tools`)**
   - 크림 배경
   - PUBLIC 라벨
   - 5개 도구 카드 + 1개 플레이스홀더 카드
   - 업데이트 예고 문구 ("손을 더 봐야 하는 것들이 책상 위에 몇 개 놓여 있는데…")
   - 사용 전 안내 4개 항목 (notice)

4. **섹션 3 — 교내 대시보드 (`#dashboard`)**
   - 베이지 배경
   - INTERNAL 라벨
   - 타이틀: "실제 운영 중인 온라인 교무실"
   - `<details>` 아코디언 (기본 접힘 상태)
   - 펼치면: 기술 안내 박스 + 8개 기능 칩 + 4개 모의 화면 탭(홈/게시판/학사일정/연락망)
   - 모의 화면은 **모든 데이터가 ○ 로 마스킹** — 학교 정보 노출 0%

5. **섹션 4 — 닫는 섹션 (`#about`)**
   - 크림 배경
   - 테두리 없는 탁 트인 마무리
   - 인용구 → 구분선 → 본문(피드백 요청) → 보조 링크 (Gmail / 주소 복사)
   - 뒤이어 푸터: 저작권 + 자동 갱신 날짜 (JavaScript)

## 6. 새 도구 추가 워크플로우

새 도구를 추가할 때 수정해야 할 곳:

### A. `index.html` 의 도구 카드 그리드
플레이스홀더(`.tool-card.placeholder`) **바로 앞**에 추가:

```html
<a href="새도구URL" target="_blank" rel="noopener" class="tool-card">
  <div class="tool-icon">이모지</div>
  <div class="tool-name">도구 이름</div>
  <div class="tool-desc">
    한 줄 또는 두 줄 설명 (검색 키워드 자연스럽게 포함)
  </div>
  <span class="tool-tag">바로 사용</span>
</a>
```

### B. `index.html` 의 JSON-LD 구조화 데이터 (`<head>` 내부)
`hasPart` 배열에 새 항목 추가:

```json
{
  "@type": "WebApplication",
  "name": "도구 이름",
  "url": "도구 URL",
  "applicationCategory": "EducationalApplication",
  "operatingSystem": "Web",
  "offers": { "@type": "Offer", "price": "0", "priceCurrency": "KRW" }
}
```

### C. `sitemap.xml`
새 `<url>` 항목 추가:

```xml
<url>
  <loc>새 도구 URL</loc>
  <changefreq>monthly</changefreq>
  <priority>0.8</priority>
</url>
```

### D. 사용자에게 안내
새 도구 URL을 **Google Search Console "URL 검사"** 에 입력 → 색인 요청 안내.

### 주의
- `target="_blank" rel="noopener"` 필수 (새 창 열기, 보안)
- 카드 설명에 검색 키워드를 자연스럽게 포함 (메타 keywords보다 본문이 SEO에 효과적)
- 도구가 사용자 데이터를 외부 서버에 저장한다면 카드를 추가하기 전 신중히 검토 (예: 연수 등록부 도구 — 보류 중)

## 7. 작업일지 글 추가 워크플로우

### A. 새 HTML 파일 생성: `notes/YYYY-MM-DD-슬러그.html`

기존 글(`2026-04-24-opening.html`)을 템플릿으로 복사 후 수정:
- `<title>`, `<meta description>`, `og:*` 태그 갱신
- `<meta property="article:published_time">` 날짜 갱신
- `<meta property="article:section">` 카테고리 (개발/학교생활/일상)
- 본문 (`<article class="article-body">`) 교체
- 카테고리 뱃지 클래스: `dev` / `school` / `life`
- 발행일 표시 (`2026.MM.DD.` 형식)
- 이전 글 링크 (`post-nav`) — 첫 글이 아니면 이전 글 URL 연결

### B. `notes/index.html` 의 포스트 카드 리스트에 추가
**최신순(맨 위)** 으로 카드 삽입:

```html
<a href="YYYY-MM-DD-슬러그.html" class="post-card" data-category="dev|school|life">
  <div class="post-meta">
    <span class="post-category dev|school|life">🛠 개발 | 🏫 학교생활 | 📝 일상</span>
    <span>YYYY.MM.DD.</span>
  </div>
  <h2 class="post-title">제목</h2>
  <p class="post-excerpt">발췌문 (2~3줄)</p>
</a>
```

### C. `sitemap.xml` 에 새 글 URL 추가
`changefreq monthly`, `priority 0.7`

### D. 이전 글의 "다음 글" 링크 갱신 (있다면)
이전 포스트 HTML의 `.post-nav` 영역에서 "다음 글" `<span class="disabled">` 을 `<a>` 로 변경하고 새 글 링크 연결.

### 카테고리·이모지 매핑
| 카테고리 | data-category | 아이콘 | 뱃지 색상 |
|---|---|---|---|
| 개발 | `dev` | 🛠 | 파랑 (#e8edf5/#2d4f7a) |
| 학교생활 | `school` | 🏫 | 초록 (#eef5ee/#2d7a3e) |
| 일상 | `life` | 📝 | 베이지 (#f5f1e8/#8a6a45) |

## 8. 배포 워크플로우 (Git)

이 저장소는 GitHub Pages로 배포됩니다. 모든 변경사항은 `main` 브랜치에 푸시되어야 자동 반영됩니다.

### Bash 스크립트 패턴 (Claude가 사용)

```bash
cd /tmp && rm -rf jerry_repo && \
git clone https://github.com/jerry5026-dotcom/jerry5026-dotcom.github.io.git jerry_repo 2>&1 | tail -1 && \
cp "/c/Users/User/Desktop/클로드작업/깃허브 페이지/index.html" /tmp/jerry_repo/index.html && \
cd /tmp/jerry_repo && \
git add index.html && \
git commit -m "커밋 메시지" && \
git push origin main 2>&1 | tail -3 && \
cd /tmp && rm -rf jerry_repo
```

### 인증
- 로컬 git에 `jerry5026-dotcom` 계정 + Credential Manager 설정 완료
- 별도 인증 불필요

### 배포 시간
- 푸시 후 GitHub Pages 반영까지 **1~2분** 소요
- 사용자에게 항상 안내할 것
